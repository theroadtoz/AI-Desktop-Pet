>
> 核验日期：2026-06-09
>
> 原则：事实优先使用 Steam、GitHub、产品官网和官方监管资料。动态数据必须标注日期；未找到可靠证据的判断标记为”待验证假设”。

## 1. 核心结论

“Live2D + AI 对话 + 桌面常驻”已经不是市场空白，也不能再使用“第一”“唯一”作为定位。

当前市场处于早期密集竞争阶段。基础能力正在快速同质化，项目可验证的机会主要位于：

1. 长期驻留时的低资源占用。
2. 不打扰工作的主动交互策略。
3. Windows 窗口吸附、穿透、多显示器和睡眠恢复。
4. 角色动作编排、人格稳定性和官方角色质量。
5. 清晰、真实的本地与云端隐私边界。

## 4. 需要废弃的旧结论

以下结论缺乏证据或已被直接竞品证伪：

- “高桌面集成 + 高 AI 能力象限完全空白。”
- “第一个真正能聊天的 Live2D 桌面宠物。”
- “唯一同时做到真桌宠、Live2D、AI 和零门槛。”
- “开源方案拒绝 99% 的潜在用户。”
- “18 个月技术窗口。”
- “某产品以约 18 元上架，因此买断制已经验证。”
- “3D 角色陪伴感天然低于 Live2D。”
- “Character.AI 付费率低于 0.5%、年推理成本超过 2 亿美元。”除非补充可靠来源和日期。

## 5. 商业判断

### 已有证据

- AI 伴侣和桌宠两类需求都存在。
- AI 桌面伙伴已经有多款商业和开源产品。
- 市场收入可能高度集中，单纯进入品类不等于获得商业成功。
- 云端模型具有持续推理成本；低价买断不能自然覆盖无限云端用量。

### 待验证假设

- 用户是否愿意为更低资源、更低打扰的桌面伙伴付费。
- `¥18-36`是否是合适的首发价格。
- 用户是否接受 BYOK。
- 用户是否愿意为角色包、语音或云端高级模型持续付费。
- 本地推理带来的隐私优势是否足以抵消下载体积和硬件要求。

## 6. 推荐比较维度

后续矩阵应只比较可观察或可测试的指标，不使用主观 10 分制给未开发产品打目标分。

| 维度 | 建议测量方式 |
|---|---|
| 首次使用门槛 | 从安装到首次成功对话的步骤和时间 |
| 常驻资源 | 固定硬件下的 CPU、GPU、内存和耗电 |
| 桌面融合 | 穿透、拖拽、吸附、多显示器、DPI、睡眠恢复测试 |
| 角色表现 | 统一对话脚本下的表情正确率和动作自然度盲测 |
| AI 体验 | 首 token 延迟、人设一致性、错误率和中断恢复 |
| 隐私 | 默认数据流、第三方接收者、本地模式能力 |
| 商业表现 | 价格、评价量、评价结构；销量只能使用明确标注的估算 |
| 留存代理 | 更新频率、社区活跃、评论中的长期使用反馈 |
